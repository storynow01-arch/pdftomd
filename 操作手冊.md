# 📅 行事曆解析系統 - 操作手冊

本系統能將學校或單位的「PDF 格式行事曆」轉換為結構化的 Markdown 資料，並提供一鍵同步至 Notion 知識庫與 Google 行事曆的自動化流程。

## 🚀 系統特色

*   **強大解析能力**：使用 Google 最新 `gemini-2.5-flash` 模型，支援掃描檔、複雜表格、圖片排版等各式 PDF 文件。
*   **單位篩選**：自動辨識並分類行事曆上的單位（如教務處、學務處），支援複選寫入。
*   **無縫整合**：直接串接 Notion Database 與 Google Calendar，省去人工登錄的時間。
*   **智能降級與防錯機制**：在 API 忙碌時具有 Fallback 防呆機制，並在介面上提示使用者。

---

## 🛠️ 操作流程說明

### 第一步：上傳 PDF 文件
1. 進入系統首頁。
2. 將包含行事曆內容的 PDF 檔案 **拖曳** 至畫面的虛線框內，或點擊框內 **「📥 拖曳 PDF 到這裡，或點擊選擇檔案」**。
3. 系統最大支援 10MB 的 PDF 檔案。
4. 選擇檔案後，點擊左側的 **「開始解析 →」** 按鈕。
5. 留意下方的百分比進度條（約需 10~30 秒），系統完成後將自動導向預覽頁面。

### 第二步：MD 預覽與校對 (Preview)
此步驟讓您在寫入資料庫前，檢視並確認 AI 解析的準確性。

*   **單位頁籤切換**：上方會列出所有解析出的單位（例如：教務處、學務處）。您可以點擊切換查看特定單位的事件。
*   **單位複選功能**：您可以點擊多個單位進行「複選」（已選的單位右上角會顯示綠色的 `✓` 角標）。**只有被選取的單位才會在下一步被寫入 Notion。** 若要寫入所有內容，請點選最前方的「📋 全部」即可清空選取，回復至全選模式。
*   **關鍵字篩選**：利用放大鏡圖示的搜尋框輸入關鍵字（如「段考」、「會議」），可快速過濾出包含該字眼的事件。
*   **編輯模式**：當您**只選取單一單位**時，可點擊「✏️ 編輯」按鈕進入編輯模式，手動修正 AI 辨識錯誤的文字或日期。修正完成後請切換回「👁 預覽」。
*   **寫入 Notion**：確認無誤後，點擊右下角浮動的主按鈕 **「寫入 Notion →」**。系統會將您選取的單位資料寫入 Notion 頁面。
    *   *註：如果您不想寫入 Notion，可以點擊次要按鈕「跳過 Notion →」直接進入行事曆步驟。*

### 第三步：加入 Google 行事曆 (Calendar)
此步驟可將解析出的事件同步至您的 Google 行事曆。

*   **事件檢視**：畫面會列出所有解析出的事件（此頁面不受上一步的單位複選限制，會顯示全部事件供您挑選）。
*   **選取事件**：勾選您想要加入 Google 行事曆的事件。您也可以利用上方單位的頁籤篩選，並點選「本頁全選」或「本頁取消」來快速大量勾選。
*   **執行同步**：點擊右下角綠色的 **「📅 加入 N 個事件」** 按鈕，系統便會透過 Google Apps Script 將這些事件同步到您的行事曆中。
*   **完成**：畫面上方會顯示綠色的成功提示訊息，代表事件已成功建立。

## ⚠️ 常見問題與注意事項

1.  **為什麼有些事件沒有被解析出來？**
    如果 PDF 排版過於複雜或跨頁截斷，AI 可能會遺漏。請在「預覽頁」中利用「單一單位編輯」功能手動補上，或者確保上傳的 PDF 檔案畫質清晰且無不正常的水印遮擋。
2.  **出現「⚠️ 主模型配額暫時耗盡...」的警告怎麼辦？**
    這代表主模型 `gemini-2.5-flash` 目前請求量過大被限速，系統暫時切換使用了備用模型進行解析。備用模型可能會導致解析品質稍降（例如地點顯示「無」或漏掉特定事件）。建議您稍待 1~2 分鐘後重新上傳解析，以確保最佳的資料品質。
3.  **寫入 Notion 後，我想修改內容怎麼辦？**
    寫入 Notion 是一次性的操作，若需修改，建議直接前往您的 Notion 頁面中進行調整；或是返回系統首頁重新解析，修改後再次寫入（這將會建立一個新的 Notion 頁面）。

## 🔧 Google Apps Script (GAS) 部署教學

為了讓系統能夠將事件寫入您的 Google 行事曆，我們需要建立一個中繼的 GAS Web API。

### 步驟 1：建立 GAS 腳本
1. 前往 [Google Apps Script 首頁](https://script.google.com)。
2. 點擊左上角的 **「新增專案」**。
3. 將專案名稱改為：`行事曆解析系統 API`。
4. 清空預設的程式碼，並將以下完整的 `calendar.gs` 程式碼複製貼上：

```javascript
// ============================================================
// Google Apps Script - Calendar 操作腳本
// 複製此程式碼到 script.google.com 並部署為 Web App
// ============================================================

/**
 * 處理 POST 請求（來自 Next.js 後端）
 * 支援的 action：addEvent, deleteEvent, listEvents
 */
function doPost(e) {
  try {
    const data = JSON.parse(e.postData.contents);
    
    // 驗證密鑰（從環境變數讀取，與 Next.js .env.local 一致）
    const SECRET_KEY = PropertiesService.getScriptProperties().getProperty('SECRET_KEY');
    if (SECRET_KEY && data.secretKey !== SECRET_KEY) {
      return buildResponse(403, { error: '驗證失敗' });
    }

    const action = data.action;

    if (action === 'addEvent') {
      return handleAddEvent(data.event);
    } else if (action === 'addEvents') {
      return handleAddEvents(data.events);
    } else if (action === 'deleteEvent') {
      return handleDeleteEvent(data.eventId);
    } else if (action === 'listEvents') {
      return handleListEvents(data.startDate, data.endDate);
    } else {
      return buildResponse(400, { error: '未知的 action: ' + action });
    }
  } catch (err) {
    return buildResponse(500, { error: err.message });
  }
}

/**
 * 新增單一行事曆事件
 */
function handleAddEvent(event) {
  if (!event) return buildResponse(400, { error: '缺少 event 資料' });

  const calendar = CalendarApp.getDefaultCalendar();
  
  const startTime = new Date(event.startTime);
  const endTime = new Date(event.endTime);
  
  let calEvent;
  if (event.isAllDay) {
    calEvent = calendar.createAllDayEvent(
      event.title,
      startTime,
      { description: event.description || '', location: event.location || '' }
    );
  } else {
    calEvent = calendar.createEvent(
      event.title,
      startTime,
      endTime,
      { description: event.description || '', location: event.location || '' }
    );
  }

  return buildResponse(200, {
    success: true,
    eventId: calEvent.getId(),
    title: calEvent.getTitle(),
    message: '事件已成功加入 Google 行事曆'
  });
}

/**
 * 批次新增多個事件
 */
function handleAddEvents(events) {
  if (!events || !Array.isArray(events)) {
    return buildResponse(400, { error: '缺少 events 陣列' });
  }

  const results = [];
  const errors = [];

  events.forEach(function(event, index) {
    try {
      const response = handleAddEvent(event);
      const result = JSON.parse(response.getContent());
      results.push({ index: index, ...result });
    } catch (err) {
      errors.push({ index: index, error: err.message });
    }
  });

  return buildResponse(200, {
    success: true,
    added: results.length,
    errors: errors,
    results: results
  });
}

/**
 * 刪除行事曆事件
 */
function handleDeleteEvent(eventId) {
  if (!eventId) return buildResponse(400, { error: '缺少 eventId' });

  const calendar = CalendarApp.getDefaultCalendar();
  const event = calendar.getEventById(eventId);
  
  if (!event) {
    return buildResponse(404, { error: '找不到指定事件' });
  }

  event.deleteEvent();
  return buildResponse(200, { success: true, message: '事件已刪除' });
}

/**
 * 列出指定時間範圍的事件
 */
function handleListEvents(startDate, endDate) {
  const calendar = CalendarApp.getDefaultCalendar();
  const start = startDate ? new Date(startDate) : new Date();
  const end = endDate ? new Date(endDate) : new Date(start.getTime() + 30 * 24 * 60 * 60 * 1000);
  
  const events = calendar.getEvents(start, end);
  
  const eventList = events.map(function(event) {
    return {
      id: event.getId(),
      title: event.getTitle(),
      startTime: event.getStartTime().toISOString(),
      endTime: event.getEndTime().toISOString(),
      description: event.getDescription(),
      location: event.getLocation()
    };
  });

  return buildResponse(200, { success: true, events: eventList });
}

/**
 * 建構 JSON 回應
 */
function buildResponse(statusCode, data) {
  const output = ContentService.createTextOutput(JSON.stringify(data));
  output.setMimeType(ContentService.MimeType.JSON);
  return output;
}

// ============================================================
// 設定密鑰（在 GAS 編輯器執行此函數一次即可）
// 把 YOUR_SECRET_KEY 換成和系統環境變數裡 GAS_SECRET_KEY 一樣的值
// ============================================================
function setupSecretKey() {
  PropertiesService.getScriptProperties().setProperty('SECRET_KEY', 'YOUR_SECRET_KEY');
  Logger.log('密鑰設定完成');
}
```
</details>

### 步驟 2：設定密鑰 (建議)
1. 在 GAS 編輯器中找到 `setupSecretKey` 函數（在程式碼最下方）。
2. 將 `'YOUR_SECRET_KEY'` 換成您自訂的一串密碼（例如 `'my_secure_calendar_key_2026'`）。
3. 點擊編輯器上方的「執行」按鈕，選擇 `setupSecretKey` 函數執行。
4. 第一次執行會要求授權您的 Google 帳號，點擊「審查權限」➔ 選擇帳號 ➔ 「進階」➔ 「前往...」➔「允許」。

### 步驟 3：部署為 Web 應用程式
1. 點擊右上角藍色的 **「部署」** ➔ **「新增部署」**。
2. 點擊齒輪圖示 ⚙️，選擇 **「Web 應用程式」**。
3. 設定：
   *   **說明**：`行事曆 API v1`
   *   **執行身分**：`我（您的 Gmail 帳號）`
   *   **誰可以存取**：`所有人` *(必須設為所有人，由上方的密鑰保護安全性)*
4. 點擊「部署」。
5. **複製顯示出來的「網頁應用程式網址 (Web App URL)」**。您稍後部署 Vercel 時會需要它。

## 🌐 Vercel 部署教學

本系統是基於 Next.js 框架開發，最適合且最簡單的部署方式是使用 Vercel。

### 步驟 1：準備 GitHub 儲存庫
1. 將本專案的所有程式碼推送到您的 GitHub 儲存庫（Private Repo 即可）。
   *⚠️ 警告：千萬不要將 `.env.local` 檔案推送上去，裡面包含您的私密金鑰。*

### 步驟 2：在 Vercel 建立專案
1. 登入 [Vercel](https://vercel.com/) (可使用 GitHub 帳號登入)。
2. 點擊 **「Add New...」** ➔ **「Project」**。
3. 找到您的 GitHub 儲存庫，點擊 **「Import」**。
4. Framework Preset 會自動偵測為 `Next.js`，保持預設即可。

### 步驟 3：設定環境變數 (Environment Variables)
在點擊 Deploy 之前，展開 **「Environment Variables」** 區塊，並將您系統所需的環境變數逐一填入。本系統需要以下變數：

| Key 名稱 | Value 說明 |
| :--- | :--- |
| `GEMINI_API_KEY` | 來自 [Google AI Studio](https://aistudio.google.com/app/apikey) 的 Gemini API 金鑰。負責 PDF 解析核心功能。 |
| `NOTION_TOKEN` | Notion 的 Internal Integration Secret Token，通常以 `ntn_` 或 `secret_` 開頭。 |
| `NOTION_PAGE_ID` | 要寫入的 Notion 頁面 ID (32 碼字串，不含橫線)。 |
| `GAS_CALENDAR_URL` | 上一個階段中，您部署 Google Apps Script 所獲得的 Web App URL。 |
| `GAS_SECRET_KEY` | *(選填)* 若您在 GAS 中有執行設定密鑰，請在此填寫一樣的密鑰。 |

### 步驟 4：執行部署
1. 變數設定完成後，點擊 **「Deploy」**。
2. Vercel 會自動進行建置 (Build) 流程，約需 1~2 分鐘。
3. 建置完成後，您會獲得一個專屬的網址（例如 `your-project.vercel.app`），點擊該網址即可開始使用您的雲端行事曆解析系統！
